# R27.1a — Bugfix Sweep Changelog

**Base:** R27.1 (commit fe67007)
**Type:** Single-commit bugfix sweep (9 bugs + admin sidebar addition)
**Scope:** Additive only — no tables dropped/renamed, no endpoints removed/renamed, hash routing preserved.

---

## BUG 1 — Order confirmation email not delivered
- Added `[boot] brevo configured: yes/no` log at server boot (`server/index.ts`).
- Order-confirm email in `POST /api/shop/orders` now passes `text:` fallback + `event: "shop_order_confirm"` and logs failures as `[email] order_confirm failed:`.
- Wired **signup welcome email** (sent post-verify in `/api/shop/verify-otp`) via `sendShopWelcomeEmail` (event `shop_welcome`).
- Wired **order status update emails** for `confirmed | packed | dispatched | delivered` in `PATCH /api/admin/shop-orders/:id/status` (event `shop_order_${status}`).
- All email sends are fire-and-forget and logged to the notification log.

## BUG 2 — Email OTP verification on signup (STRICT MODE)
- Migration `R27.1a` adds to `shop_users`: `email_verified`, `verify_otp`, `verify_otp_expires_at`, `verify_otp_sent_at` (existing users grandfathered to `email_verified=1`... see note below).
- `createShopUser` now creates accounts with `email_verified=0`, a hashed 6-digit OTP (10-min expiry), and returns `{ user, otp }`.
- Signup (`POST /api/shop/auth/signup`) **no longer issues a token**; returns `{ verify_required: true, email }` and emails the OTP.
- Login (`POST /api/shop/auth/login`) returns **403 `{ error: "verify_required", email }`** if the account is unverified (and re-sends a fresh OTP).
- New endpoints:
  - `POST /api/shop/verify-otp` — `{ email, otp }` → on success sets `email_verified=1`, clears OTP fields, issues a session token, sends the welcome email.
  - `POST /api/shop/resend-otp` — `{ email }` → rate-limited (max 3/hour per account).
- New page `/#/customer/verify` (6-digit input, Resend with 60s cooldown, email pre-filled from query).

## BUG 3 — Customer session not persisting
- `shop-auth.tsx` `setAuth`/`clear` now dispatch `window.dispatchEvent(new Event("shop:auth-changed"))`.
- Context subscribes to both `storage` and the custom `shop:auth-changed` event to re-sync the token across tabs/components.
- Checkout redirect now carries `?next=checkout`; login/verify return the user to checkout on success.

## BUG 4 — Admin Web Customers detail view
- `GET /api/admin/shop-customers/:id` now returns joined `orders`, `addresses`, and `wishlist` (wishlist was previously missing) plus `emailVerified`.
- New page `/#/admin/web-customers/:id` (alias `/#/admin/customer-users/:id`): profile card (email, phone, email-verified status, joined, last login), Addresses / Orders / Wishlist tables. Order rows link to the order detail.
- Web Customers list rows are now clickable through to the detail page.

## BUG 5 — Currency picker (₹/$)
- Already shipped in R27.1 (`CurrencyPicker` / `HeaderActions` in `SiteLayout.tsx`, `client/src/lib/currency.ts`). Verified present and functional. No code change.

## BUG 6 — PO Dashboard "Process PO" button
- New route `POST /api/admin/purchase-orders/:id/mark-processed` (reuses `v2.processPurchaseOrder`): splits confirmed lines → `processed`; returns 400 when no confirmed lines / PO not found, 404 when the PO does not exist.
- `AdminPODetailV2` header now shows a **Process PO** button (or a "Processed" badge once processed); refreshes detail + list on success.

## BUG 7 — PO Dashboard inline search + date picker
- `GET /api/admin/purchase-orders-v2` extended with `q`, `from`, `to` (YYYY-MM-DD; inclusive day bounds).
- `AdminPOs` adds From/To date pickers next to the existing debounced search box, with a Clear button.

## BUG 8 — Freight Charges search
- Already shipped in R27.1 (`GET /api/admin/freight-charges?q=` filters by part number / product name). Verified. No code change.

## BUG 9 — Duplicate PO / Duplicate Quotation
- Existing routes `POST /api/admin/purchase-orders/:id/duplicate` and `POST /api/admin/quotations/:id/duplicate` (date reset to today) surfaced in the UI:
  - `AdminPOs`: Duplicate (copy icon) row action → navigates to the new PO detail.
  - `AdminQuotes`: Duplicate (copy icon) row action → opens the copy in the Team portal.

## Admin sidebar — Web Orders tab

**Changed endpoints:** `POST /api/shop/auth/signup` (no token, returns verify_required), `POST /api/shop/auth/login` (403 verify_required path), `POST /api/shop/orders` (email event/text), `PATCH /api/admin/shop-orders/:id/status` (status emails), `GET /api/admin/shop-customers/:id` (+wishlist), `GET /api/admin/purchase-orders-v2` (+q/from/to).
**New columns (shop_users):** `email_verified`, `verify_otp`, `verify_otp_expires_at`, `verify_otp_sent_at`.
**New pages:** `/#/customer/verify`, `/#/admin/web-customers/:id` (+ `/admin/customer-users/:id` alias).
