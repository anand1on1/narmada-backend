# R27.1 — Docket Fix + E-commerce Phase 1

## Summary
Fixes the Consignment Portal "From Delhi" docket display, and adds a customer-facing
e-commerce layer (cart, customer accounts, COD checkout, order tracking) plus admin
tooling (web orders, web customers, freight charges), a currency picker with live FX,
duplicate PO/quotation, and chat attachment columns.

All migrations are additive and idempotent (`[migrations] R27.1: ...` markers). No
existing table was dropped or renamed. No existing endpoint was removed or renamed.

## TASK 0 — Docket fix (From Delhi)
- Delhi PO list + PO detail modal now surface `docket_url` and `docket_number`
  resolved from the `dispatches` table, so "View Docket" buttons are enabled and the
  Docket # no longer shows "—".

## TASK 1 — Cart + Buy Now
- `client/src/lib/cart.ts` — localStorage cart (`narmada_cart`) with pub/sub + cross-tab sync.
- Product page: quantity stepper, **Add to Cart** and **Buy Now**.
- `/#/cart` page with qty edit / remove / subtotal.
- Header cart icon with live count badge.

## TASK 2 — Customer accounts (email + password)
- `client/src/lib/shop-auth.tsx` — `ShopAuthProvider` / `useShopAuth` / `shopFetch` (x-shop-token).
- Pages: `/#/customer/login`, `/#/customer/signup`, `/#/customer/account`,
  `/#/customer/orders`, `/#/customer/orders/:id`, `/#/customer/wishlist`.
- Saved addresses CRUD + wishlist.
- Passwords hashed with scrypt (`salt:hash`); sessions in `shop_sessions`.

## TASK 3 — Checkout (COD) + emails + auto-PO
- `/#/checkout` — COD only, saved-address select or new, live freight quote.
- On order: confirmation email to customer (Brevo SMTP, fire-and-forget) + auto-PO
  notification to procurement (fire-and-forget). Order number `NM/ORD/YY/NNNN`.

## TASK 4 — Admin web orders
- `/#/admin/orders` list (status filter + search) → `/#/admin/orders/:id` detail with
  status workflow (placed→confirmed→processing→dispatched→delivered / cancelled),
  status history, and dispatch carrier/docket capture.

## TASK 5 — Admin web customers
- `/#/admin/web-customers` — search + sort (recent / spend), order count + total spend.

## TASK 6 — Freight charges manager
- `/#/admin/freight` — per part_number freight, search, zero-only filter, inline edit,
  add/update, CSV import (`part_number,freight_inr`). Quote engine multiplies per-unit × qty.

## TASK 7 — Currency picker + live FX
- `client/src/lib/currency.ts` — INR/USD, `narmada_currency` persisted.
- `GET /api/public/currencies` returns rate (falls back to 83.5 if FX not yet fetched).
- Header dropdown switches displayed prices site-wide.

## TASK 8 — Duplicate PO + duplicate quotation
- `POST /api/admin/purchase-orders/:id/duplicate`
- `POST /api/admin/quotations/:id/duplicate`
- Clones line items with dates reset to today.

## TASK 9 — Chat attachments (PARTIAL — see open items)
- Migration columns `customer_chat_messages.attachment_url` + `attachment_type` added.
- **Deferred**: the AiSensy attachment send helper + the send-failure investigation +
  the compose-attachment UI are NOT implemented in this release. See R27.1-AUDIT-REPORT.md.

## New tables (all `CREATE TABLE IF NOT EXISTS`)
shop_users, shop_sessions, shop_addresses, shop_wishlist, shop_orders,
shop_order_items, shop_order_status_history, freight_charges, shop_settings.

## New columns
customer_chat_messages.attachment_url, customer_chat_messages.attachment_type.
