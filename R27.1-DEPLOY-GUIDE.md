# R27.1 — Deploy Guide

## Backend (Render — narmada-backend.onrender.com)
1. Merge/push `main` (this R27.1 commit). Render auto-builds (`npm run build`) and
   restarts (`npm start` → `node dist/index.cjs`).
2. On boot, idempotent migrations run automatically. Confirm in Render logs:
   - `[migrations] R27.1: complete`
   - `[boot] step: post-R27.1 migrations`
   - `[boot] listening on PORT ...`
3. No manual SQL needed. Migrations are additive `CREATE TABLE IF NOT EXISTS` /
   `ALTER ... ADD COLUMN` wrapped per-statement in try/catch — safe to re-run.
4. SQLite data dir on Render: `/opt/render/project/src/data` (unchanged).

### Optional env
- Brevo SMTP must be configured for order-confirmation + auto-PO emails (already used
  by existing notifications). If absent, order still places (emails are fire-and-forget).
- FX: `/api/public/currencies` falls back to 83.5 INR/USD until the daily fetch populates
  a live rate. No action required.

## Frontend (GoDaddy static hosting)
1. Deliverable: `narmada-mobility-godaddy.zip` (built from `dist/public`, no source maps).
2. Unzip to the GoDaddy web root, replacing the previous build. Keep `.htaccess`
   (included in the zip) for hash-routing + asset caching.
3. Frontend talks to `https://narmada-backend.onrender.com` (config.js unchanged).

## Smoke test after deploy
- Public: open a product → Add to Cart → cart badge increments → `/#/cart` shows item.
- Currency dropdown toggles INR/USD prices.
- `/#/customer/signup` → login → `/#/checkout` (COD) → order confirmation page.
- Admin: `/#/admin/orders` shows the order; status + dispatch update persist.
- Admin: `/#/admin/freight` add a part + CSV import; checkout freight reflects it.
- Consignment "From Delhi": View Docket buttons enabled; PO modal shows Docket #.

## Rollback
Revert the R27.1 commit and redeploy. New tables are inert if unused; no destructive
change to existing tables, so an older build runs unaffected against the same DB.
